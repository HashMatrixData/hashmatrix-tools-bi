# DataEase 集成边界（接入约定占位）

> **状态：占位（约定先行，实现待落地）**。本文定义 tools-bi 与 **DataEase** 的集成边界与约束，
> 不含任何客户/环境可识别信息（红线合规）。具体接线随后续 Issue 落地。

## 为什么集成而非自研

重型 BI 报表设计器（70+ 图表、自助分析、仪表板编排）**集成 DataEase**，而非自研报表设计器。
tools-bi 提供**后端取数基座与集成边界**；大屏/图表**编辑器前端在 `webui`（AntV）**，本仓不做编辑器前端。

## 三条集成约定（北极星对齐）

### 1. SSO —— 统一身份，单点登录

- 身份源 = Keycloak **Organizations 单 realm**（`org` = 租户，JWT 带 `tenant` 声明），与平台一致。
- DataEase 作为 **OIDC 客户端**接入该 realm；用户在平台登录后免再登录进入 DataEase。
- **不**在 DataEase 内自建用户体系；账号/组织映射以 Keycloak 为准。

### 2. 租户上下文透传 —— 隔离取数

- 平台请求经网关注入 `X-Tenant-*` 头（架构 05 §5），到 tools-bi 由 `starter-tenant` 落 `TenantContext`。
- DataEase 的**数据源按租户隔离**：每租户取数指向其 Doris 取数库 `bi_<tenant>`（见 `TenantCatalog`），
  由 `control-plane` 编排开通；报表/仪表板查询绝不跨租户串。
- 集成层须把当前租户上下文透传至 DataEase 的数据源选择与查询边界（嵌入 URL / 反代头 / 会话三选一，随实现确定）。

### 3. 部署级品牌外框 —— 不按租户换肤

- 品牌是**部署级**（部署期配置注入）：公网 SaaS 用统一我们品牌；私有化用客户品牌（部署级）。
- DataEase 以**部署级品牌外框**嵌入（平台壳 + 品牌化外观），**不按租户在运行期动态换肤**。
- 嵌入方式（iframe 嵌入 / 反向代理 / 主题注入）在落地 Issue 中定稿；外框品牌取自部署期配置，与运行期租户无关。

## 边界（本仓做 / 不做）

- **做**：取数基座（Doris 连通 + 租户隔离取数路由）、集成边界约定、租户上下文透传到集成产品。
- **不做**：自研报表设计器；编辑器前端（在 `webui`）；DataEase 自身的部署与运维（在 `deploy`/部署层）。
